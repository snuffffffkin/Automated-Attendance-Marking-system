# Face Recognition Attendance System — Full Breakdown

## Libraries

**`cv2` (OpenCV)** — Computer vision library. Handles camera feeds, drawing rectangles/text on frames, and displaying video windows.

**`face_recognition`** — Built on top of `dlib`. Does the heavy lifting: loading face images, generating numerical "encodings" (vectors) for faces, and comparing them.

**`os`** — File system operations: checking if paths exist, listing directories, joining paths.

**`numpy (np)`** — Numerical computing. Used here for `np.argmin()` — finding the index of the smallest value in an array.

**`csv`** — Reading and writing `.csv` files row by row.

**`datetime`** — Getting current date/time and formatting/parsing it.

**`time`** — Imported but not actually used in this code.

## CONFIG Section

```python
KNOWN_FACES_DIR = "known_faces"
```
Folder where reference photos are stored. Expected structure:
```
known_faces/
  John/
    photo1.jpg
    photo2.jpg
  Sarah/
    photo1.jpg
```
Each subfolder name becomes the person's recognized name.

```python
ATTENDANCE_FILE = "attendance.csv"
EVENTS_FILE = "attendance_events.csv"
```
Two separate CSVs. `EVENTS_FILE` is a raw log of every entry/exit event. `ATTENDANCE_FILE` is the final computed result (Present/Absent).

```python
TOLERANCE = 0.38
```
The threshold for face matching. Face recognition works by computing a **distance** between two face encodings — a float between 0 and 1. Lower = more similar. If the distance is below `0.38`, it's considered a match. The default in the library is `0.6`. This value (`0.38`) is very strict — reduces false positives but may miss real matches if photos are low quality.

```python
FRAME_SCALE = 0.25
```
Each frame is shrunk to 25% of its original size before processing. Face detection is slow — this makes it 16× faster (0.25² = 1/16 the pixels). Faces are then scaled back up for display.

```python
CLASS_DAY = datetime.now().strftime("%A")
CLASS_START = "15:00"
CLASS_END = "16:00"
```
`strftime("%A")` returns the full weekday name, e.g. `"Monday"`. So `CLASS_DAY` is set to **today** when the script starts. Attendance is only recorded between 15:00 and 16:00.

```python
URL_IN = "http://10.108.142.166:8080/video"
URL_OUT = "http://10.108.142.199:8081/video"
```
IP camera streams (e.g. from an Android phone running an IP Webcam app). OpenCV can open these URLs just like a local camera.

```python
known_encodings = []
known_names = []
```
Two parallel lists. Index `i` in `known_encodings` always corresponds to index `i` in `known_names`. So if `known_names[3] == "John"`, then `known_encodings[3]` is John's face vector.

## Functions

### `setup_files()`
```python
def setup_files():
    if not os.path.exists(EVENTS_FILE) or os.path.getsize(EVENTS_FILE) == 0:
        with open(EVENTS_FILE, "w", newline="") as f:
            writer = csv.writer(f)
            writer.writerow(["name", "date", "time", "event", "camera"])
```
Creates the CSV files **only if they don't exist or are empty**. `newline=""` is required on Windows to prevent blank rows between entries. `writer.writerow(...)` writes the header row. Same logic repeats for `ATTENDANCE_FILE`.

---

### `is_class_time()`
```python
def is_class_time():
    now = datetime.now()
    current_day = now.strftime("%A")
    current_time = now.strftime("%H:%M")
    return current_day == CLASS_DAY and CLASS_START <= current_time <= CLASS_END
```
Gets current day and time as strings. The comparison `CLASS_START <= current_time <= CLASS_END` works because strings like `"15:00"` and `"15:45"` compare correctly in alphabetical/lexicographic order — this only works reliably with zero-padded `HH:MM` format.

---

### `already_recorded_recently(name, event, seconds=10)`
```python
def already_recorded_recently(name, event, seconds=10):
    if not os.path.exists(EVENTS_FILE):
        return False
    now = datetime.now()
    with open(EVENTS_FILE, "r", newline="") as f:
        reader = csv.DictReader(f)
        for row in reader:
            if row["name"] == name and row["event"] == event:
                row_time = datetime.strptime(
                    row["date"] + " " + row["time"], "%Y-%m-%d %H:%M:%S"
                )
                if (now - row_time).total_seconds() < seconds:
                    return True
    return False
```
Prevents spam-logging. Since the camera processes ~30 frames/second, the same face would be logged thousands of times without this check.

`csv.DictReader` reads each row as a dictionary where keys are column headers — so `row["name"]` instead of `row[0]`.

`datetime.strptime(...)` parses a string into a `datetime` object using a format pattern. Here it combines `"2025-01-15"` + `" "` + `"14:32:10"` into one datetime.

`(now - row_time)` produces a `timedelta` object. `.total_seconds()` converts it to a plain number. If a matching event was logged less than 10 seconds ago, return `True` (already recorded).

---

### `record_event(name, event, camera)`
```python
def record_event(name, event, camera):
    if not is_class_time():
        return
    if already_recorded_recently(name, event, seconds=10):
        return
    now = datetime.now()
    today = now.strftime("%Y-%m-%d")
    current_time = now.strftime("%H:%M:%S")
    with open(EVENTS_FILE, "a", newline="") as f:
        writer = csv.writer(f)
        writer.writerow([name, today, current_time, event, camera])
    print(f"{event} recorded: {name} at {current_time}")
```
Two guards at the top — exits immediately if it's not class time, or if this event was logged recently. Then opens the file in append mode (`"a"`) to add a new row without overwriting existing data. `event` will be either `"IN"` or `"OUT"` depending on which camera detected the person.

---

### `load_known_faces()`
```python
def load_known_faces():
    for person_name in os.listdir(KNOWN_FACES_DIR):
        person_path = os.path.join(KNOWN_FACES_DIR, person_name)
        if os.path.isdir(person_path):
            for filename in os.listdir(person_path):
                if filename.lower().endswith((".jpg", ".jpeg", ".png")):
                    image_path = os.path.join(person_path, filename)
                    image = face_recognition.load_image_file(image_path)
                    encodings = face_recognition.face_encodings(image)
                    if encodings:
                        known_encodings.append(encodings[0])
                        known_names.append(person_name)
```
`os.listdir()` returns a list of names inside a directory. `os.path.join()` safely concatenates path parts with the right separator (`/` or `\`). `os.path.isdir()` skips any stray files in the root folder.

`face_recognition.load_image_file()` reads the image as a NumPy array in RGB format (note: OpenCV uses BGR by default, but this library uses RGB).

`face_recognition.face_encodings(image)` returns a **list** of 128-dimensional vectors — one per face found. It returns a list because there could be 0 or multiple faces in one photo. Only the first encoding (`[0]`) is used. If the image has no detectable face, `encodings` is empty and the image is skipped.

---

### `process_frame(frame, camera_name, event_type)`
This is the core of the system, called on every video frame.

```python
small_frame = cv2.resize(frame, (0, 0), fx=FRAME_SCALE, fy=FRAME_SCALE)
rgb_small_frame = cv2.cvtColor(small_frame, cv2.COLOR_BGR2RGB)
```
Resizes to 25%. Then converts from BGR (OpenCV default) to RGB (face_recognition requirement). This is a critical step — without it, colors are interpreted incorrectly and recognition fails.

```python
face_locations = face_recognition.face_locations(rgb_small_frame)
face_encodings = face_recognition.face_encodings(rgb_small_frame, face_locations)
```
`face_locations` returns a list of tuples `(top, right, bottom, left)` — bounding boxes for every face found. Passing `face_locations` into `face_encodings` is an optimization: it skips the detection step since locations are already known.

```python
distances = face_recognition.face_distance(known_encodings, face_encoding)
```
Computes the Euclidean distance between the detected face encoding and **every** encoding in `known_encodings`. Returns an array of floats, one per known person.

```python
best_index = np.argmin(distances)
best_distance = distances[best_index]
if best_distance < TOLERANCE:
    name = known_names[best_index]
```
`np.argmin()` finds the index of the smallest distance — the closest match. If that distance is below the tolerance threshold, the name at that index is used. Otherwise the person stays `"Unknown"`.

```python
scale = int(1 / FRAME_SCALE)  # = 4
top *= scale
right *= scale
bottom *= scale
left *= scale
```
The bounding box coordinates are from the small frame, so they're multiplied back by 4 to map correctly onto the original full-size frame.

```python
color = (0, 255, 0) if name != "Unknown" else (0, 0, 255)
cv2.rectangle(frame, (left, top), (right, bottom), color, 2)
cv2.rectangle(frame, (left, bottom - 35), (right, bottom), color, cv2.FILLED)
cv2.putText(frame, name, (left + 6, bottom - 6), cv2.FONT_HERSHEY_SIMPLEX, 0.8, (0,0,0), 2)
```
OpenCV colors are `(Blue, Green, Red)`. Green `(0,255,0)` for known, red `(0,0,255)` for unknown. First `rectangle` draws the face box outline. Second draws a filled label background behind the name. `putText` writes the name in black (`(0,0,0)`) over that background.

### `calculate_attendance()`
```python
for person in set(known_names):
    final_status[person] = "Absent"
```
`set(known_names)` removes duplicates (since the same person may have multiple photos loaded). Everyone starts as Absent.

```python
for name, events in events_by_person.items():
    events.sort(key=lambda x: x["time"])
    last_event = events[-1]["event"]
    if last_event == "IN":
        final_status[name] = "Present"
    elif last_event == "OUT":
        final_status[name] = "Absent"
```
Groups all of today's events per person, sorts them chronologically, and checks the **last** event. Logic: if the last thing they did was enter (`IN`), they're Present. If they left (`OUT`), they're Absent. This handles cases like someone briefly stepping out and coming back.

## Main Loop

```python
cap_in = cv2.VideoCapture(URL_IN)
cap_out = cv2.VideoCapture(URL_OUT)
```
`VideoCapture` accepts either an integer (local webcam index) or a string (URL/file path).

```python
if not cap_out.isOpened():
    cap_out = None
```
Graceful fallback — if the exit camera is unavailable, the system still runs on the entrance camera only.

```python
while True:
    ret_in, frame_in = cap_in.read()
```
`cap.read()` returns two values: `ret` (bool — whether the frame was read successfully) and `frame` (the NumPy array of pixel data). The loop runs until `q` is pressed.

```python
if cv2.waitKey(1) & 0xFF == ord("q"):
    break
```
`waitKey(1)` waits 1 millisecond for a key press and returns the key code. `& 0xFF` masks to the last 8 bits (for cross-platform compatibility). `ord("q")` is the ASCII code for `q` (113).
